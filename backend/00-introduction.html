2. 트랜잭션 & 동시성 기초

트랜잭션 정상 / 롤백 흐름

sequenceDiagram participant Proxy as Spring Proxy participant Svc as Service participant DB as Database Note over Proxy,DB: 정상 흐름 Proxy->>DB: BEGIN TRANSACTION Proxy->>Svc: 메서드 호출 Svc->>DB: INSERT / UPDATE DB-->>Svc: OK Svc-->>Proxy: 반환 Proxy->>DB: COMMIT Note over Proxy,DB: 예외 발생 시 Proxy->>DB: BEGIN TRANSACTION Proxy->>Svc: 메서드 호출 Svc->>DB: INSERT Svc->>Svc: RuntimeException 발생 Svc-->>Proxy: 예외 전파 Proxy->>DB: ROLLBACK
@Transactional 프록시 동작 — Spring AOP 프록시가 트랜잭션을 감싼다

동시성 처리 전략 비교

전략방식장점단점사용 시기
낙관적 락
@Version		버전 컬럼으로 충돌 감지, 충돌 시 재시도락 없음, 높은 처리량충돌 많으면 재시도 폭증충돌 적을 때 (카탈로그 업데이트)
비관적 락
FOR UPDATE		읽기 시 즉시 락 획득직관적, 확실한 보장락 대기, 데드락 위험충돌 많을 때 (재고 차감)
원자적 UPDATEUPDATE SET qty=qty-1 WHERE qty>=1단순, 락 없음조건 복잡해지면 관리 어려움단순 증감 연산 권장
Redis 원자 연산DECR, Lua 스크립트초고 TPS 가능DB와 일관성 유지 필요콘서트 티켓, 쿠폰 발급
🎯 면접 포인트 — @Transactional 동작 안 하는 케이스
  1. 자기 호출(Self-invocation): 같은 클래스 내 메서드 → 프록시 우회
  2. private 메서드: Spring AOP 프록시 불가
  3. @Async 조합: 별도 스레드 → 트랜잭션 컨텍스트 없음
  4. RuntimeException 이 아닌 Checked Exception: 기본 rollback 안 됨
  5. 트랜잭션 내 외부 HTTP 호출: 분산 트랜잭션 환상 🔥 Deep-dive

3. API 설계 원칙

원칙올바른 예잘못된 예
리소스는 명사GET /orders/{id}GET /getOrder?id=1
HTTP 메서드 의미 준수DELETE /orders/{id}POST /orders/delete/{id}
상태 코드 정확성생성: 201, 없음: 404, 검증 실패: 400모든 오류에 200 반환
에러 포맷 (RFC 7807){"type","title","status","detail"}에러마다 다른 포맷
멱등성 키Idempotency-Key 헤더 지원중복 요청 처리 없음
페이지네이션Cursor/Keyset 방식OFFSET 방식 (깊어질수록 느림)

에러 응답 표준 (RFC 7807 Problem Details)

{
  "type":   "https://example.com/errors/out-of-stock",
  "title":  "재고 부족",
  "status": 409,
  "detail": "SKU sku-1234 의 가용 재고가 0입니다.",
  "traceId": "4bf9e7b2-..."
}
모든 에러 응답을 동일한 스키마로 통일 → 클라이언트 에러 핸들링 단순화

6. 자주 나오는 함정 (면접 필수)

#함정증상해결책
1트랜잭션 내 외부 HTTP/Kafka 호출부분 성공/실패, 이벤트 유실Outbox 패턴 🔥
2@Transactional 자기 호출트랜잭션 미적용별도 Bean으로 분리
3낙관적 락 없이 재고 차감Lost update, 오버셀@Version 또는 원자적 UPDATE
4Retry 무한루프Retry storm → 시스템 다운Exponential backoff + 최대 횟수
5Timeout 미설정스레드 풀 고갈, 연쇄 장애모든 외부 호출에 Timeout 명시
6N+1 쿼리페이지 로드 느림Fetch join / @EntityGraph / Batch size
7로그에 PII/토큰 노출보안 사고마스킹 필터, 구조화 로그 검증
8H2로 통합 테스트Mock은 통과, 프로덕션 실패Testcontainers
