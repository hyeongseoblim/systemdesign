2. 시간·공간 복잡도

Big-O(빅오)는 입력 크기 n이 커질 때 알고리즘의 최악 성능 상한을 나타낸다.

O(1)       < O(log n)   < O(n)     < O(n log n) < O(n²)    < O(2ⁿ)
상수 시간    로그 시간     선형 시간    선형 로그     이차 시간    지수 시간

예시:
O(1)       : HashMap 조회, 배열 인덱스 접근
O(log n)   : 이진 탐색, B+Tree 인덱스 탐색
O(n)       : 배열 순차 탐색, 단순 루프
O(n log n) : Merge Sort, Quick Sort (평균)
O(n²)      : 버블 정렬, 중첩 루프
O(2ⁿ)      : 완전 탐색 (부분집합 열거)

면접 함정
"HashMap은 O(1)이죠?" — 평균은 O(1)이지만 해시 충돌이 집중되면 최악 O(n). Java 8+에서 Bucket 크기 8 초과 시 LinkedList → Red-Black Tree로 전환해 O(log n)으로 완화. 초기 capacity와 load factor(0.75 기본) 설정으로 리사이징 최소화.

Context Switch(문맥 교환) 비용

Context Switch는 CPU가 실행 중인 작업을 바꾸는 과정이다. PCB(Process Control Block, 프로세스 제어 블록)에 레지스터, 프로그램 카운터, 스택 포인터를 저장하고 다음 작업의 상태를 복구한다.

  • 직접 비용: 레지스터 저장/복구, 커널 진입/복귀
  • 간접 비용: CPU 캐시(L1/L2) 무효화 → 캐시 미스 증가
  • TLB flush: 프로세스 전환 시 페이지 테이블 캐시 초기화
스레드가 항상 빠르지 않은 이유
스레드 수가 CPU 코어 수를 초과하면 Context Switch 비용이 증가한다. 동기화(Mutex/Lock) 경쟁이 심할 경우 Lock Contention(락 경합)으로 오히려 단일 스레드보다 느려질 수 있다 (Amdahl's Law).

가상 메모리(Virtual Memory) & 페이징(Paging)

각 프로세스는 자신이 전체 메모리를 독점한다고 착각하는 가상 주소 공간을 갖는다. MMU(Memory Management Unit)가 가상 주소 → 물리 주소로 변환. TLB(Translation Lookaside Buffer, 변환 참조 버퍼)는 최근 변환 결과를 캐싱해 속도를 높인다.

면접 질문
"Java의 Heap과 OS의 Heap은 같은 건가요?" — 다르다. JVM Heap은 OS Heap 위에 할당된 별도 메모리 풀이다. JVM이 OS로부터 큰 블록(-Xmx)을 미리 받아서 객체를 배치하고, GC(Garbage Collection, 가비지 컬렉션)가 관리한다.

"TIME_WAIT가 왜 필요한가요?" — ① 마지막 ACK가 유실될 경우 서버의 FIN 재전송에 응답하기 위해, ② 네트워크에 떠다니는 지연 패킷이 새 연결에서 오판되지 않도록. 기간은 2×MSL(Maximum Segment Lifetime, 최대 세그먼트 수명, 보통 60~120초). TIME_WAIT가 너무 많으면 소켓 포트 고갈 → SO_REUSEADDR 또는 연결 풀링으로 해결.

Deadlock(교착 상태) 4 조건

Deadlock은 다음 4가지 조건이 동시에 성립할 때 발생한다. 하나라도 제거하면 예방 가능.

Deadlock 4 조건 — 4가지 조건 중 하나를 제거하면 교착 상태 예방 가능
Deadlock 방지 전략
  • 순환 대기 제거: 모든 스레드가 동일한 순서로 락 획득 (가장 실용적)
  • 타임아웃: tryLock(timeout)으로 일정 시간 내 획득 실패 시 포기
  • 락 개수 최소화: 여러 락을 하나의 더 큰 락으로 통합
  • Lock-free 자료구조: CAS 기반으로 락 없이 원자적 연산

Java Memory Model(JMM, 자바 메모리 모델) & happens-before

happens-before 관계: A happens-before B이면, A의 결과가 B에 반드시 보임을 보장한다.

  • 동일 스레드 내 순서
  • volatile 변수 쓰기 → 읽기
  • 모니터 unlock → lock
  • Thread.start() 호출
  • Thread.join() 반환
