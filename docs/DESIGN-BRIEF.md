# jobStudy — 디자인 브리프 (Design Brief)

> 이 문서는 **디자인 업무를 착수하기 위한 컨텍스트 패키지**다. 디자이너/디자인 에이전트는 이 문서 하나로 제품·현재 상태·제약·작업 범위를 파악할 수 있어야 한다. 코드 근거는 `apps/web/` 에 있다.

---

## 1. 제품 컨텍스트

| 항목 | 내용 |
|---|---|
| **제품** | 백엔드 이직 준비용 **학습 카드 플랫폼** (모바일 우선 PWA) |
| **타깃 사용자** | 6년차 백엔드 개발자 1인 (시니어 이직 준비). 출퇴근·자투리 시간에 폰으로 카드 학습 |
| **핵심 경험** | ① 영역별 카드 피드를 훑고 → ② 카드 상세(개념·다이어그램·표·코드)를 읽고 → ③ 이해도 확인 질문에 답을 적는다 |
| **콘텐츠** | DB의 Card. 수동 47장 + AI 생성이 매일 누적. 본문은 **마크다운(Mermaid·표·코드·callout 포함)** |
| **톤&무드(현재 추정)** | 집중·차분·기술적. 화려함보다 가독성·정보 밀도. 다크 우선 |
| **플랫폼** | Next.js 15(App Router, RSC) + React 19 PWA. 폰 세로 화면이 1순위 |

**한 줄 요약**: "매일 쌓이는 백엔드/시스템 디자인 학습 카드를 폰에서 편하게 읽고 답을 적는 앱."

## 2. 현재 화면 · 컴포넌트 인벤토리

### 화면 (2개)
1. **피드** (`app/page.tsx`) — 상단바(타이틀+부제) · 영역 필터 칩(가로 스크롤) · 카드 리스트 · "더 보기" 버튼 · 빈/에러 상태
2. **카드 상세** (`app/cards/[id]/page.tsx`) — 뒤로가기 · 메타(영역/코치/난이도) · 제목 · **마크다운 본문**(Mermaid·표·코드·리스트) · 이해도 Q&A(textarea)

### 컴포넌트 (CSS 클래스 기준, `app/globals.css`)
`.topbar` · `.tabs`/`.tab` · `.feed` · `.card`(`.meta`·`.badge`·`.badge.mode`·`.diff`·`.summary`·`.tags`/`.tag`) · `.loadmore` · `.empty` · `.detail` · `.content`(마크다운) · `.mermaid-box` · `.qsection`/`.qcard`/`textarea`

## 3. 현재 디자인 시스템 스냅샷 (as-is)

**스타일링**: 순수 CSS + CSS 변수. **Tailwind·CSS-in-JS 없음.** 라이트/다크는 `prefers-color-scheme` 자동 전환(수동 토글 없음).

### 컬러 토큰 (`:root`)
| 토큰 | Dark(기본) | Light |
|---|---|---|
| `--bg` | `#0f172a` | `#f8fafc` |
| `--card` | `#1e293b` | `#ffffff` |
| `--card-hover` | `#334155` | `#f1f5f9` |
| `--text` | `#e2e8f0` | `#0f172a` |
| `--muted` | `#94a3b8` | `#64748b` |
| `--accent` | `#38bdf8` (sky) | `#0284c7` |
| `--border` | `#334155` | `#e2e8f0` |
| `--radius` | `14px` | — |

팔레트 계열: **Slate(뉴트럴) + Sky(액센트)**. Tailwind slate/sky 색과 동일 계열.

### 타이포그래피
- 폰트: system UI 스택 + `Noto Sans KR` (별도 웹폰트 로드 없음)
- 본문 15px / line-height 1.7. 스케일: 피드 제목 16.5px · 상세 h1 22px · 본문 h1 20 / h2 18 / h3 16 · 배지 11px · 메타 12–13px
- **정식 타입 스케일·간격(spacing) 토큰은 아직 없음** (px 하드코딩 산재)

### 레이아웃
- 컨테이너 `max-width: 720px`, 좌우 패딩 16px, 하단 96px. 상단바 sticky + backdrop blur.

## 4. 기술 제약 (디자인 시 반드시 준수)

1. **마크다운 본문을 견뎌야 함** — 카드 본문은 임의의 마크다운이다. 디자인은 다음 요소 전부를 우아하게 렌더해야 한다: 제목(h1–h4)·문단·리스트·**표**·인라인/블록 **코드(kotlin/sql 등)**·**blockquote(=callout: 면접포인트/함정/팁)**·**Mermaid 다이어그램**. 특정 콘텐츠에 맞춘 디자인 금지(범용이어야 함).
2. **Mermaid** — 클라이언트에서 `mermaid.render`로 SVG 생성(현재 테마 `neutral`). 다크 배경과 대비·브랜드 정합성을 디자이너가 정의해야 함(`.mermaid-box`).
3. **한국어 우선 타이포** — 본문 다수가 한글+영문 기술용어 혼용. 한글 가독성(자간·행간·폰트) 중요.
4. **PWA/모바일** — 세로 폰 1순위, `standalone` 실행, `theme_color`/상태바 색 존재. 터치 타깃 ≥ 44px 권장.
5. **성능·단순성** — 무거운 UI 라이브러리·대용량 웹폰트 지양. 순수 CSS 유지 선호(도입 시 근거 필요).
6. **접근성** — 라이트/다크 양쪽에서 WCAG AA 대비. 현재 `--muted` 계열의 소형 텍스트 대비 점검 필요.

## 6. 디자인 작업 백로그 (우선순위별)

> 각 항목은 독립적으로 착수 가능. `[P1]`=높음.

### P1 — 기반 & 브랜드
- **[P1] 디자인 토큰 정식화** — 컬러(현행 유지/개선) + **타입 스케일** + **spacing 스케일**을 CSS 변수로 체계화. 하드코딩 px 제거 기준 제시.
- **[P1] 영역(Area) 컬러/아이콘 시스템** — 7개 영역 각각의 색+아이콘 정의(배지·필터칩·카드 좌측 인디케이터에 적용). 다크/라이트 양쪽 대비 확보.
- **[P1] 모드(mode) 시각 구분** — 개념/설계/면접/리뷰를 배지·아이콘으로 즉시 구분.
- **[P1] PWA 앱 아이콘 제작** — `public/icon-192.png`·`icon-512.png` **현재 없음**(manifest는 참조 중). 마스커블 아이콘 포함.

### P2 — 화면 완성도
- **[P2] 카드(피드 아이템) 리디자인** — 정보 위계(영역·모드·난이도·제목·요약·태그) 정리, 난이도 1–5 시각화(점/바), 터치 피드백.
- **[P2] 카드 상세 리딩 경험** — 마크다운 타이포 리듬, **callout(blockquote) 스타일**(면접포인트=강조/함정=경고/팁=긍정), 코드블록·표·Mermaid 박스 스타일 통일.
- **[P2] 상태 디자인** — 로딩(스켈레톤), 빈 상태, 에러(“API 연결 실패”) , "더 보기"/무한스크롤.
- **[P2] Q&A 영역** — 답변 입력·저장 상태(현재 저장 UI 미약), 답변 완료 표시, (향후 서버 동기화 대비 상태 슬롯).

### P3 — 확장
- **[P3] 라이트/다크 수동 토글** + 시스템 연동.
- **[P3] 학습 진도/북마크 UI** (데이터 모델에 `interactions.bookmarked` 존재).
- **[P3] 하단 네비/제스처** 등 앱다운 내비게이션, 상단바 스크롤 반응.
- **[P3] Mermaid 브랜드 테마** — 노드/엣지 색을 토큰과 정합.

## 7. 산출물 · 핸드오프 규칙

- **최종 반영 위치**: 디자인 토큰·컴포넌트 스타일은 `apps/web/app/globals.css`(+필요 시 컴포넌트 지역 스타일). 순수 CSS 유지 원칙.
- **컴포넌트 매핑**: 시안은 위 §2 컴포넌트/클래스명에 매핑해 전달(개발 반영 용이).
- **상태 명세**: 각 컴포넌트의 default/active/hover/pressed/disabled/loading/empty/error 를 함께.
- **반응형**: 폰(≤430px) 우선, 태블릿(720px 컨테이너 상한)까지.
- **접근성**: 라이트·다크 각각 대비비(AA) 명시.
- **미리보기**: `cd apps/web && npm run dev` → `http://localhost:3000` (백엔드 8080 필요). 백엔드 없이 볼 땐 목 데이터로 스크린 확인.

## 8. 결정 필요 사항 (디자이너가 정할 열린 질문)

1. **브랜드 방향** — 현행 Slate+Sky를 다듬을지, 새 팔레트로 리브랜딩할지?
2. **다크/라이트 우선순위** — 다크 기본 유지? 라이트도 1급 지원?
3. **정보 밀도** — 카드에 요약·태그를 다 노출(밀도↑) vs 간결(밀도↓)?
4. **영역 표현** — 색만? 색+아이콘? 아이콘 스타일(라인/솔리드)?
5. **난이도 표현** — 숫자(현행 "난이도 3/5") vs 점/막대/라벨?
