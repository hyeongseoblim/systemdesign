4. 데이터 모델

erDiagram CARD ||--o{ CARD_QUESTION : has CARD ||--o{ INTERACTION : tracked_by CURRICULUM_TOPIC ||--o| CARD : generates CARD { uuid id PK string area "system-design, logistics..." string coach "생성 코치 페르소나" string mode "concept/design/interview/review" string title string slug text summary "피드 카드 요약" text content_md "마크다운 본문" int difficulty "1-5" string_array tags string status "draft/published/archived" int quality_score "자가검증 0-100" string source "ai_generated/manual" timestamp created_at timestamp published_at } CARD_QUESTION { uuid id PK uuid card_id FK text question int order } CURRICULUM_TOPIC { uuid id PK string area string title int order bool generated date planned_date } INTERACTION { uuid id PK uuid card_id FK text answer "Q&A 답변" bool bookmarked timestamp last_viewed_at }
ERD — CARD가 중심. CURRICULUM_TOPIC이 "무엇을 생성할지" 관리, INTERACTION이 학습 진행 기록
설계 의도
  • CURRICULUM_TOPIC: 품질 게이트 1(중복 방지)의 핵심. 미리 주제를 커리큘럼으로 정의 → 매일 generated=false 중 하나를 골라 생성
  • content_md: HTML이 아닌 마크다운으로 저장 → 프론트에서 렌더 + 다이어그램은 mermaid 코드블록으로
  • quality_score: 자가검증 통과 점수. 임계치 미달이면 status=draft로 보류 (사용자에게 안 보임)
  • INTERACTION: 기존 localStorage+Worker 동기화를 DB로 정식 흡수

5. AI 생성 파이프라인 + 품질 게이트 3종

sequenceDiagram participant C as ⏰ Scheduler participant W as Card Generator participant DB as PostgreSQL participant AI as Claude API C->>W: 매일 09:00 트리거 W->>DB: 오늘 생성할 주제 조회 Note over W,DB: 🚪 Gate1 — 중복 방지
generated=false 중 N개만 W->>DB: 일일 생성 수/토큰 누적 확인 Note over W: 🚪 Gate3 — 예산 캡
상한 초과 시 중단 loop 주제별 W->>AI: 카드 생성 (코치 페르소나 프롬프트) AI-->>W: 카드 초안 (구조화 JSON) W->>AI: 2차 호출 — 사실/구조 검증 AI-->>W: quality_score + 지적사항 Note over W: 🚪 Gate2 — 자가검증
점수 ≥ 70 → published
미달 → draft 보류 W->>DB: 카드 저장 + 토픽 generated=true end
생성 파이프라인 — 3개 게이트로 중복·품질·비용을 자동 통제
🚪 품질 게이트 3종
  1. 중복 방지: CURRICULUM_TOPIC에서 generated=false인 주제만 생성 대상. 같은 주제 두 번 생성 불가
  2. 자가 검증: 생성 직후 LLM 2차 호출로 사실성·구조·난이도 검증 → quality_score. 70점 미만은 draft로 보류(사용자 미노출)
  3. 예산 캡: 일일 생성 카드 수 상한(예: 3개) + 누적 토큰 한도. 초과 시 당일 생성 중단 → 비용 폭주 차단
⚠️ 1A의 본질적 리스크
완전 자동 게시는 틀린 학습 자료를 만들 수 있다. 자가 검증(Gate2)은 환각을 100% 막지 못하므로, 향후 draft 카드를 모바일에서 빠르게 승인/반려하는 경량 검수 화면을 옵션으로 둔다(하이브리드로 후퇴 가능한 안전판).
