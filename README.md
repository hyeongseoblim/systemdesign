# SystemDesign

시스템 디자인 학습 및 정리 저장소입니다. 대규모 시스템 설계에서 자주 등장하는 개념, 패턴, 사례 연구(case study)를 주제별로 기록합니다.

## 목표

- 시스템 디자인의 핵심 개념을 꾸준히 정리한다.
- 실제 서비스 사례를 분석하여 트레이드오프를 이해한다.
- 인터뷰 대비용 설계 문제 풀이 과정을 기록한다.

## 다루는 주제

- **기초**: 확장성, 가용성, 신뢰성, 일관성, 지연 시간
- **네트워크**: HTTP/HTTPS, DNS, CDN, 로드 밸런싱
- **데이터 저장소**: RDBMS, NoSQL, 캐시, 샤딩, 복제
- **메시징**: 메시지 큐, 이벤트 스트리밍, Pub/Sub
- **아키텍처 패턴**: 마이크로서비스, 이벤트 주도, CQRS
- **사례 연구**: URL 단축기, 뉴스 피드, 채팅, 검색 등
